# 商家运营动作闭环设计

## 1. 目标

本次增强的目标是让商家端不再停留于“只看分析结果”，而是支持商家根据热销商品、冷门商品和异常风险直接执行运营动作，形成“分析结果 -> 经营建议 -> 运营动作”的闭环。

## 2. 范围

本次只实现四个固定动作：

1. 设为重点观察
2. 加入促销
3. 补货优先
4. 优化详情页

本次不实现真实促销系统、不实现库存扣减、不实现订单流转，只做动作记录、动作状态展示和动作汇总。

## 3. 页面设计

### 3.1 商家端动作入口

动作按钮放在 `MerchantDashboard.vue` 的三个区域中：

- 热销机会：`设为重点观察`、`补货优先`
- 滞销风险：`优化详情页`
- 冷门商品：`加入促销`

### 3.2 动作状态反馈

点击动作后，商品卡片内显示状态标签，例如：

- 已设为重点观察
- 已加入促销
- 已标记补货优先
- 已记录详情页优化

### 3.3 动作汇总

商家页新增一个小卡片：

- 标题：`今日已执行运营动作`

显示当天四类动作的计数和总动作数，用于证明系统具备运营闭环。

## 4. 后端设计

### 4.1 数据模型

新增 `MerchantAction` 表，保存：

- `merchant_id`
- `product_id`
- `action_type`
- `status`
- `created_at`

### 4.2 接口

新增两个接口：

- `POST /api/strategy/actions`
- `GET /api/strategy/actions/summary`

其中：

- POST 用于记录动作
- GET 用于返回今日动作统计和已执行动作列表

### 4.3 权限

两个接口都只允许 `merchant` 角色访问。

## 5. 前端数据流

1. 商家页加载时请求动作汇总
2. 点击按钮后调用动作记录接口
3. 接口成功后更新本地动作状态
4. 页面显示状态标签和动作汇总

## 6. 测试要求

### 6.1 后端

需要验证：

- 商家可以成功记录动作
- 动作汇总能返回计数结果
- 非商家用户被拒绝访问

### 6.2 前端

需要验证：

- 商家页出现动作按钮
- 点击按钮后触发接口调用
- 页面显示已执行状态
- 页面显示动作汇总卡片

## 7. 价值

完成后，系统将从“日志分析看板”升级为“支持商家执行运营动作的智能分析与决策支持系统”，更符合老师对“有交互、有分析结果、有落地动作”的要求。
