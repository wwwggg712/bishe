## 背景与目标

当前“今日经营简报”区域展示为长文本 +（曾经的）右侧经营盈亏块，用户希望：

1) 经营盈亏（收入/成本/利润-亏损）应放到“经营总览小卡片”中，商家第一眼看到赚钱亏钱。
2) 今日经营简报的标题需要“框起来”，视觉更聚焦。
3) 全链路禁止出现 `P0/P1/...` 等优先级字母标签；输出与展示均使用分点分段，且前端具备强制清洗能力（即使模型偶尔输出也不会展示）。

## 范围

- 前端：`MerchantDashboard` 的经营总览卡片区与今日经营简报卡片区改版；`MerchantOpsOverview` 支持可选隐藏其自带的盈亏小卡片，避免重复。
- 后端：LLM system prompt 继续明确禁止 `P0/P1`，确保生成内容倾向分点输出。
- 不新增接口，不改数据库。

## 数据口径

- 收入/成本/利润沿用 `fetchMerchantOpsOverview(days=30)` 返回的 `summary.revenue/cost/profit`（购买次数×售价/成本价；利润=收入-成本）。
- 盈亏标签：profit < 0 时显示“亏损”，否则显示“利润”。

## UI 设计

### 经营盈亏 → 经营总览小卡片

- 在 `MerchantDashboard` 的经营总览小卡片区（原“行为总量/浏览量/成交笔数/转化率”）前追加三张卡：
  - 近{days}天收入
  - 近{days}天成本
  - 近{days}天利润/亏损
- 布局使用三列卡片（`merchant-dashboard__summary--triple`），第一行即可看到“收入/成本/利润(亏损)”。

### 今日经营简报 → 单列 + 标题框化

- 移除简报卡片右侧区域（不再左右两栏），避免无意义留白。
- “今日经营简报”标题增加框/胶囊样式（与全站标题风格一致），作为视觉锚点。
- 简报正文保持“核心结论/机会点/风险预警/行动清单”分段分点展示。

### 禁止 P0/P1

- 后端 system prompt 明确：行动清单只用 `1）2）3）`，禁止 `P0/P1/P2...`。
- 前端展示层再做一次强制清洗：
  - 支持剥离 `P0-` / `P1-3）` / `P2)` 等各种前缀变体；
  - 即使模型不听话，页面也不会出现 P 字母优先级。

## 验收标准

- 经营盈亏在经营总览小卡片区首行可见，右侧不再单独占块。
- 今日经营简报标题“框起来”，并且简报卡片为单列布局，无多余留白。
- 页面任何位置不出现 `P0/P1` 字样（包括模型输出与前端展示）。
- 前端测试全量通过；后端测试全量通过。
